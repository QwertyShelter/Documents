# 打通“远端走本地代理”的 SSH 反向转发
目标：让**远端**访问 `127.0.0.1:7890` 时，实际上走的是**你本地电脑**的 Clash 代理。

在本地开一个长期运行的 SSH 隧道（保持窗口别关）：

```bash
ssh -N -R 127.0.0.1:7890:127.0.0.1:7890 WS_USER@WS_HOST
```

解释如下：

- `-R 127.0.0.1:7890:127.0.0.1:7890` 的意思是：
  - **远端**开一个 `127.0.0.1:7890`
  - 这个端口的流量通过 SSH 回到**本地**的 `127.0.0.1:7890`（Clash Mixed Port）

> 如果你本地 Mixed Port 不是 7890，替换两处 `7890`。

### 验证远端代理是否联通

在另一个终端 SSH 上远端，执行：

```bash
curl -I -x http://127.0.0.1:7890 https://example.com
```

能返回 HTTP 头（200/301/302 等）就说明远端已经能用本地代理出网。
